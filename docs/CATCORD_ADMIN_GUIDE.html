Catcord – Admin Guide

Operational reference for Discord server administrators.

Configuration

  • Optional guild restriction via GUILD_ID in .env

Language Relay

Mirror and translate messages between channels using /langrelay commands.

Common Commands

/langrelay_group_create <name>
/langrelay_group_delete <name>
/langrelay_group_add <name> #channel CODE
/langrelay_group_remove <name> #channel
/langrelay_group_list
/langrelay_provider provider:<deepl|openai>
/langrelay_replymode state:<on|off>
/langrelay_thread_mirroring state:<on|off>
/langrelay_reaction_mirroring state:<on|off>
/langrelay_status
/langrelay_power state:<on|off>
/langrelay_group_power group:<name> state:<on|off>

Mappings persist across bot restarts and are stored per guild in ./data/langrelay/<guild_id>.json.

Permissions: bot requires Send Messages, Manage Webhooks, and Read Message History.

Auto-Translate

Enable automatic translation in a channel.

Commands

/autotranslate_on target:<code> [source:<code>] [formality:<default|less|more>] [min_chars:<number>]
/autotranslate_status
/autotranslate_off

Requires translation API key. Per-channel settings are not persistent.

Reminders

Schedule repeating messages using /reminder commands.

Syntax

/reminder add name:<id> channel:<#channel> message:<text> [headline:<title>] [interval:<number> unit:<minutes|hours|days>] [weekday:<day>] [time:<HH:MM>] [once:true]

Examples

  • /reminder add name:backup channel:#general message:"Run backup" time:02:00
  • /reminder add name:standup channel:#dev message:"Daily standup" interval:1 unit:days time:09:00
  • /reminder add name:colony channel:#ops message:"Territory Development\n-Building Power\n-Fortification Power" headline:"Colony Action" interval:1 unit:days
  • /reminder add name:launch channel:#ops message:"Launch prep" time:13:00 once:true
  • /reminder remove name:backup
  • /reminder list
  • /reminder toggle enabled:false

Reminders persist across bot restarts and are stored per guild in ./data/reminder/<guild_id>.json. When a time is provided, reminders fire on the minute (UTC). Supplying only a time schedules a daily reminder, and adding once:true creates a one-time reminder that deletes itself after sending. Provide an optional headline to place the reminder inside an embed, and include \n in the message text for multi-line reminders.

Pause reminder delivery for a server with /reminder toggle enabled:false; re-enable with /reminder toggle enabled:true. Reminders remain saved while delivery is disabled.

Permissions: members need Use Application Commands to create or remove reminders, and the bot must have Send Messages in the target channel.
